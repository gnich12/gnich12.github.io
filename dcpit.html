Disney Consumer Products

Undergraduate Associate Program Analyst

Internship | September 2008 - February 2009

Skills

Java, J2EE, Spring, Hibernate, HMTL, CSS, XML, Ajax Confluence, Toad Websphere CVS version control

Description

Disney Consumer Products is the business segment of the Walt Disney Company that is in charge of further extended the Disney brand. I was able to join their Undergraduate Associate Program (UGA), which is a 6 month paid internship program, from the period of September 2008 to February 2009. During the duration of this program, I worked at their Information Technology division in Glendale, CA where I had the opportunity to work with senior program analyst on diverse projects that ranged from programming, documenting, testing among others projects.

Accomplistments

Programming

Resource Planning - is an in house web application that keeps tracks of how the employees distribute their working hours through different assigned projects. This application was develop with J2EE technologies, Spring Framework, Hibernate, MVC design pattern and Oracle as the database. My duties in this project were to analyze, research, and generate a solution to the different bug problems that the users submit through their support system. Some of these bug problems were related to the front-end of the application and others were related to the back-end of the application. Also, with the direction of senior programmers I assisted with the design of new features for the application.

For example, we developed a script that lets the user login to the application by checking the user class; hence, principal, administrator or manager, once login, the application will display the webpages that correspond with user permission. In addition to it, I created a small class that keeps track of the users that had permissions to modify the code at the back end of the application.

Dashboard – is a web-application that monitors all of the servers that Disney has in their network. It works by performing a simple test to the servers, if there is a problem with them, it will show a different color stage alert on the website and it will send an email to alert administrators about the problem. The dashboard application is develop using mainly Java, XML, and Spring Framework.

For this project I was in charge of updating the information that will be display on the front end of the application by contacting administrators in charge of the servers to acquire the proper information. If a new dashboard page was required. I had to coordinate with the different administrators to gather all the information that will be implemented which some of them were just simple test cases such as queries to know the status of the database or server. Also the implementation of a graphical display, modeled on previous dashboards, which is basically a tree structure textbox, with the current status of the website in real time, which will change color to yellow as a warning or to red if it didn’t pass the test.

Documentation

  • Created the correspondent documentation for different features that were applied to web applications during different release versions and updated support manuals
  • Gathered statistical information to develop a report that shows the performance of the support team in handling support calls, this report included the time it took to solve and issue and how satisfied were the callers.
  • Assisted with the authoring of new content for their internal wiki site, which some of the topics were
    • Tutorials for different programming tools, and programming language
    • Status report from manager meetings
    • Contact information of the team

Testing

  • Executed test script that test the functionality of the Artwork web application to verify if there is a bug in the application. For example, if there are no login problems, the users can access the correct pages, and the correct information can be retrieved.
  • Developed functionality test cases using Excel, basically a table where I described how to proceed with the test, for example I describe a set of instructions to see if the tester can get the same results, if not the tester can input their findings.
